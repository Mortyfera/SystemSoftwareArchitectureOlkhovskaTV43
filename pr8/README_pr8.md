**Задача 1**  
Чи може виклик count \= write(fd, buffer, nbytes); повернути в змінній count значення, відмінне від nbytes? Якщо так, то чому? Наведіть робочий приклад програми, яка демонструє вашу відповідь.  
**Компіляція програми**  
gcc \-Wall pr81.c \-o pr81  
**Запуск програми**  
./pr81  
**Результат роботи програми**  
Requested: 65536, Written: 65536  
write: Resource temporarily unavailable  
**Пояснення**  
Системний виклик write() не завжди повертає значення, що дорівнює запрошеному nbytes. Він може виконати частковий запис або повернути \-1 у разі помилки. Основними причинами такої поведінки є переповнення системних буферів, переривання виклику сигналами , фізичний брак місця на диску або робота в неблокуючому режимі. У останньому випадку, якщо місце в буфері відсутнє, write() замість очікування миттєво повертає помилку EAGAIN.

Ці теоретичні положення були підтверджені практично: розроблена програма досліджує взаємодію з анонімними каналами у неблокуючому режимі. За допомогою fcntl канал було переведено у стан O\_NONBLOCK, після чого програма намагалася заповнити його блоками по 64 КБ. Консольний вивід продемонстрував, що перший запис повністю вичерпав ресурс буфера ОС, а друга спроба прогнозовано завершилася помилкою “Resource temporarily unavailable”. Це доводить стабільність роботи алгоритму та здатність системи жорстко обмежувати запис відповідно до встановлених лімітів без блокування процесу.

**Задача 4**  
Виконайте наступну програму на мові програмування С:  
int main() {  
  int pid;  
  pid \= fork();  
  printf("%d\\n", pid);  
}  
Завершіть цю програму. Припускаючи, що виклик fork() був успішним, яким може бути результат виконання цієї програми?  
**Компіляція програми**  
gcc \-Wall pr84.c \-o pr84  
**Запуск програми**  
./pr84  
**Результат роботи програми**  
44940  
0  
**Пояснення**  
Системний виклик fork() розділяє один процес на два незалежні потоки виконання – батьківський та дочірній. Оскільки обидва процеси продовжують працювати паралельно після виклику, функція printf спрацьовує двічі:

* Батьківський процес отримує PID нащадка.  
* Дочірній процес отримує 0\.

Черговість появи цих чисел у терміналі не є фіксованою, оскільки вона залежить від планувальника операційної системи.
